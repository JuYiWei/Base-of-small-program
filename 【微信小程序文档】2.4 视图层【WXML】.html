2.4 视图层【WXML】

  • 视图层由 WXML 和 WXSS 结合展示；
  • WXML 构建页面结构；

一、数据绑定

  • WXML 中的动态数据，均来自 Page.data；
  • 数据绑定使用 Mustache 语法（双大括号）；

1. 内容

<view>{{name}}</view>

Page({
  data: {
    name: '张三'
  },
})

2. 组件属性（双引号内）

<view id="item-{{id}}">必须有大括号和双引号</view>

Page({
  data: {
    id: 0
  },
})

3. 控制属性（双引号内）

<view wx-if="{{nameFlag}}">必须有大括号和双引号</view>
Page({
  data: {
    nameFlag: true
  },
})

4. 关键字（双引号内）

<view hidden="{{false}}">必须有大括号和双引号</view>

5. {{}} 内支持运算：三元、算数、逻辑、字符串运算、数据路径运算

6. 组合：可以再看看，结合下面【列表渲染】【模板】再回头看，一下子不懂没关系

// 数组
<view wx:for="{{[zero, 1, 2, 3, 4]}}"> {{item}} </view>


// 对象
<template is="objectCombine" data="{{for: a, bar: b}}"></template>


// ... 对象展开,
<template is="objectCombine" data="{{...obj1, ...obj2, e: 5}}"></template>

二、列表渲染

1. wx:for

  • wx:for="" 绑定列表数据源
  • 数组当前项，下标变量名默认 index，变量名默认 item
<view wx:for="{{array}}">{{index}} = {{item}} </view>

Page({
  data: {
    array: [
      "a",
      "b",
      "c"
    ]
  },
})
  • 使用 wx:for-item 可以指定数组当前元素的变量名（给 item 重命名而已）
  • 使用 wx:for-index 可以指定数组当前下标的变量名（同上）
<view wx:for="{{array}}" wx:for-index="idx" wx:for-item="obj">{{idx}} = {{obj}} </view>

2. block wx:for

  • <block> 标签无实际意义，代码结构上表示一小块视图，与<view> 标签基本类似（个人感觉）；
<block wx:for="{{[1, 2, 3]}}">
  <view> {{index}}: </view>
  <view> {{item}} </view>
</block>

3. wx:key

  • 设置 id，使用 item 中的某个属性值（一般设置 唯一的 id），或者使用 *this 指向 item 本身，都需要保证唯一；(设置 key 可以优化动态列表的渲染)
<view wx:for="{{array}}" wx:key="id">{{item.id}} = {{item.name}} </view>
<view wx:for="{{array}}" wx:key="*this">{{index}} = {{item}} </view>

三、条件渲染

1. wx:if

<view wx:if="{{length > 5}}"> 1 </view>
<view wx:elif="{{length > 2}}"> 2 </view>
<view wx:else> 3 </view>

2. block wx:if

<block wx:if="{{true}}">
  <view> view1 </view>
  <view> view2 </view>
</block>

3. wx:if & hidden

  • 事件切换多时：使用 hidden，反之用 wx:if
<view hidden='{{cz_hidden_yes}}'>hidden</view>
<view hidden='{{cz_hidden_no}}'>hidden</view>

<view wx:if="{{cz_hidden_yes}}">wxif</view>
<view wx:if="{{cz_hidden_no}}">wxif</view>

四、模版

  • template 代码片段，需要设定模板名称
<template name="msgItem">
<view>
  <text> {{index}}: {{msg}} </text>
  <text> Time: {{time}} </text>
</view>
</template>
  • 使用模板
<template is="msgItem" data="{{...item}}"/>

Page({
  data: {
    item: {
      index: 0,
      msg: 'this is a template',
      time: '2016-09-15'
    }
  }
})

五、事件

1. 简单示例

<view bindtap="add"> Click me! </view>

Page({
  add: function (e) {
    console.log(e);
  }
})

2. 事件分类

  • 冒泡 和 非冒泡：事件是否向父节点传递；
类型 说明
touchstart 开始
touchmove 移动
touchcancel 取消
touchend 结束
tap 点击
longpress 长按（1.5）
longtap 长按（旧版本）
animationstart 动画开始时
animationiteration 动画一次结束时
animationend 动画完成时

其他事件都是非冒泡事件

3. 事件绑定

  • key：bind 或 catch 加上 事件类型：bindtap，catchtap
  • value：指定函数
  • bind 不阻止冒泡，catch 会阻止冒泡
<view id="outer" bindtap="handleTap1">
  outer view
  <view id="middle" catchtap="handleTap2">
    middle view
    <view id="inner" bindtap="handleTap3">
      inner view
    </view>
  </view>
</view>

// 事件顺序：3-2-1

4. 事件捕获

  • 1.5.0 开始支持
  • 顺序与冒泡相反，先捕获再执行冒泡事件
  • capture-bind：不中断捕获事件

  • capture-catch：中断捕获事件，不执行冒泡

  • 不中断

<view id="outer" bindtap="handleTap1" capture-bind:tap="handleTapa">
  outer view
  <view id="middle" bindtap="handleTap2" capture-bind:tap="handleTapb">
    middle view
    <view id="inner" bindtap="handleTap3" capture-bind:tap="handleTapc">
      inner view
    </view>
  </view>
</view>
// 事件顺序：a-b-c-3-2-1
  • 中断
<view id="outer" bindtap="handleTap1" capture-bind:tap="handleTapa">
  outer view
  <view id="middle" catchtap="handleTap2" capture-catch:tap="handleTapb">
    middle view
    <view id="inner" bindtap="handleTap3" capture-bind:tap="handleTapc">
      inner view
    </view>
  </view>
</view>
// 事件顺序：a-b

六、引用

1. import

  • import 可以使用目标文件的 template，但不会包含目标文件 import 的template

  • common.wxml

<template name="test-1">
<text>{{text}}</text>
</template>
  • page.wxml
<import src="../../common/common.wxml"/>
<template is="test-1" data="{{...item}}"/>
  • page.js
Page({
  data: {
    item: {
      text: 'asd'
      }
  },
})

2. include

  • 完整包含目标文件，相当于拷贝
